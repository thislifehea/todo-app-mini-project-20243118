# 제출 보고서 (20243118_이승혜)

## 1. 프로젝트 요약
- 프로젝트: 풀스택 Todo 앱 (React + Vite, Express + MongoDB Atlas)
- 배포: Vercel (monorepo)
- GitHub: https://github.com/thislifehea/todo-app-mini-project-20243118.git
- Vercel: https://todo-app-mini-project-20243118.vercel.app

## 2. 구현 기능 (MVP)
1. Todo 항목 추가
2. Todo 목록 조회
3. Todo 완료 체크 (체크박스)
4. Todo 삭제
5. Todo 제목 수정 (추가 구현)

## 2-1. 추가 구현 기능
- 카테고리 분류: 일반/공부/운동/일상(필터 지원)
- 마감 기한(D-day) 지정 및 마감 임박 강조
- 우선순위(importance): low/medium/high 저장 및 표시
- 정렬: 생성순 / 마감 임박순 / 중요도순
- 검색: 제목 기반 검색
- 진척도 바 (Progress Bar) + 대시보드 통계(전체/완료/남은)
- 다크 모드 토글
- 로컬스토리지 동기화: 서버 실패 시 로컬에서 데이터 복원

## 3. 기술 스택
- Frontend: React, Vite, Axios
- Backend: Node.js, Express, mongoose, cors, dotenv
- Database: MongoDB Atlas
- 배포: Vercel

## 4. 작업 과정 요약
1. `todo-app` 폴더에 `frontend`, `backend` 구조 생성
2. `backend/index.js`에 MongoDB 연결 + CRUD API 구현
3. `frontend/src/App.jsx`에 CRUD UI 개발
4. CORS, 프록시, vercel.json 설정 확인
5. `.env`에 `MONGODB_URI` 추가 후 노드 서버 정상 동작 확인
6. Github push 및 Vercel 배포 완료
7. UI 컴포넌트 분리 및 디자인 개편 (그리드 카드, 다크 모드, 진행률 표시, 필터/정렬 준비)
## 5. 어려웠던 점
- `MONGODB_URI` 환경 변수 설정 누락으로 DB 연결 실패 메시지(`uri undefined`) 발생
- `backend` 루트 엔드포인트 `/` undefined로 `Cannot GET /` 메시지
- Vercel monorepo 배포 시 프론트/백엔드 동시 경로 설정 검증 필요

## 6. AI 사용 방법
- 구현 중 발견된 문제 해결 방법과 코드 패턴, 환경 설정 체크리스트를 AI에 질의
- `App.jsx` 정리 및 “제목 수정” 기능 추가 코드 생성 지원 받음
- `.env`, `vercel.json`, CORS, MongoDB URI 관련 상세 가이드 AI로 받음

## 7. 배운 점
- `dotenv` 연결과 환경변수 키(`MONGODB_URI`) 중요성
- Vercel에서 Express API + React SPA 함께 동작시키는 방법
- 몽고 Atlas IP 허용과 사용자 인증 구조 이해
- 실제 서비스에서 민감 정보는 `.env`로 관리하고 `.gitignore` 필수
